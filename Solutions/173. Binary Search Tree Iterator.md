# Approach
## Target
* binary search tree:
```
       5
      / \
     3   8
    / \   \
   1   4   9
```
* skewed tree (according to inorder traversal)
```
          1
         /
        3
       /
      4
     /
    5
   /
  8
 /
9
```

## Reason for Storing Next Node on the Left
* Because after the current node is visited, the left node has already been visited, so it is no longer important what is on the left according to the original tree.

# Complexity
- Time complexity:
  $$O(N)$$ for building tree
  $$O(1)$$ per operation

- Space complexity:
  $$O(H)$$ because of recursion

# Code
```java []
class BSTIterator {
    TreeNode newRoot; // new root
    TreeNode it; // iterator

    private void dfs(TreeNode t) {
        if (t == null) return;

        dfs(t.left);

        it.left = t;
        it.right = null;
        it = it.left;

        dfs(t.right);
    }

    public BSTIterator(TreeNode root) {
        newRoot = new TreeNode(-1);
        it = newRoot;

        dfs(root);

        // clear the left and right nodes of tail
        it.left = null;
        it.right = null;

        // make iterator point to newRoot
        it = newRoot;
    }
    
    public int next() {
        it = it.left;
        return it.val;
    }
    
    public boolean hasNext() {
        return it.left != null;
    }
}
```
