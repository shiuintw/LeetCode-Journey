# Approach
1. Keep track of the previous element`pre` and the length of consecutive and sorted interval`len`.
2. In each iteration, if `nums[i] == pre + 1`, then `len++`; if `nums[i] != pre + 1`, then `len = 1`, which means if the interval is not consecutive and sorted, then the total length by (i)th will be 1.
3. If the length of consecutive interval is equal or larger than k`len >= k`, the max is the last element`nums[i]`, or this interval is not a consecutive and sorted subarray, which means the max is `-1`.

# Complexity
- Time complexity:
  $$O(N)$$

- Space complexity:
  $$O(N)$$

# Code
```java []
class Solution {
    public int[] resultsArray(int[] nums, int k) {
        int[] ans = new int[nums.length - k + 1];

        int pre = nums[0]; // last element
        int len = 1; // consecutive length

        // init
        for (int i = 1; i < k; i++) {
            if (nums[i] == pre + 1) len++;
            else len = 1;
            pre = nums[i];
        }
        if (len >= k) ans[0] = pre;
        else ans[0] = -1;

        // main
        for (int i = k; i < nums.length; i++) {
            if (nums[i] == pre + 1) len++;
            else len = 1;
            pre = nums[i];

            if (len >= k) ans[i - k + 1] = pre;
            else ans[i - k + 1] = -1;
        }

        return ans;
    }
}
```
