# Intuition
Given a set of integer `[10010, 01000, 00101]`. It is not a nice array if and only if there exists a pair `[a, b]` that `(a & b) != 0`.
If there exists an `a` in `nums` such that `(a & (1 << n)) != 0`, then for all `b` in nums where `b != a`, it must hold that `(b & (1 << n)) == 0`

# Approach
1. Mantain OR sum `orSum` in the interval `[f, b]`
2. Once `(nums[b] & orSum) != 0`, shrink the window by `orSum ^= nums[f++]` until `(nums[b] & orSum) == 0`.
3. Update the result with the maximum valid window size.

# Complexity
- Time complexity:
  $$O(N)$$

- Space complexity:
  $$O(1)$$

# Code
```java []
class Solution {
    public int longestNiceSubarray(int[] nums) {
        int orSum = 0;
        int res = 0;
        for (int f = 0, b = 0; b < nums.length; b++) {
            while ((orSum & nums[b]) != 0) {
                orSum ^= nums[f++];
            }
            orSum |= nums[b];
            res = Math.max(b - f + 1, res);
        }
        return res;
    }
}
```
